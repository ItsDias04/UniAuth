Документация: внешний redirect token

Flow: 1) Верификация IP по redirectRoute → 2) Генерация externalToken на пользователя → 3) Авторизация пользователя в UniAuth → 4) Получение временного code и redirect на внешний маршрут → 5) Внешняя система валидирует code и использует user payload для login/register.

Путь на фронте для авто-получения и проверки токена: /oauth2/external-redirect.

Query параметры: applicationId, redirectUri, опционально state и externalToken.

Если externalToken отсутствует или истек, компонент автоматически запросит новый, получит временный code и выполнит редирект.

Пример запроса (получить временный code и redirect)
POST /api/v1/oauth2/sso/authorize
Authorization: Bearer <access_token>
{
    "clientId": "<applicationId>",
    "redirectUri": "<redirectUri>",
    "externalToken": "<externalToken>",
    "state": "opaque-state"
}
{
    "authorizationCode": "a1b2c3...",
    "expiresInSeconds": 120,
    "redirectUrl": "<redirectUri>?code=a1b2c3...&state=opaque-state"
}
Пример запроса (валидность временного токена/code)
POST /api/v1/oauth2/sso/validate-temporary-token
{
    "authorizationCode": "a1b2c3..."
}
// valid
{
    "valid": true,
    "user": {
        "userId": "55be7f0f-e651-47f2-aaf8-a6f9e3f923ba",
        "clientId": "<applicationId>",
        "email": "john.doe@example.com",
        "firstName": "John",
        "lastName": "Doe",
        "avatarUrl": null
    }
}

// invalid
{
    "valid": false,
    "reason": "Authorization code is invalid or expired"
}
