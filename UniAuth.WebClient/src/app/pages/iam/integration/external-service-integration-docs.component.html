1. Базовые настройки интеграции

  • UniAuth API base URL: http://localhost:3001/api/v1
  • Страница bridge UniAuth (локально): http://localhost:4200/oauth2/external-redirect/{token1}
  • Если UniAuth web размещен отдельно, используйте реальный публичный URL вместо localhost:4200.
  • У внешнего сервиса должен быть зарегистрирован redirectRoute в UniAuth Developers Console.
  • Внешний сервис работает через постоянный API token приложения (Bearer), а не через JWT владельца.

3. Что должен сделать внешний сервис

  1. Добавить кнопку "Войти с помощью UniAuth" на страницу логина (Angular).
  2. Создать backend endpoint старта SSO (NestJS), который получает Token 1 и возвращает redirectUrl на UniAuth bridge.
  3. Добавить callback endpoint, который принимает token3 и вызывает introspect-token-3.
  4. На успешной валидации token3 создать/найти локального пользователя и выдать локальную сессию внешнего сервиса.
  5. На ошибке вернуть пользователя на страницу логина с понятным сообщением.
  6. Хранить application API token в секретах (env/secret manager), не в клиентском Angular-коде.

5. Чеклист проверки интеграции

  • Кнопка UniAuth видна на странице логина внешнего сервиса.
  • По кнопке фронт зовет backend start endpoint и уходит на UniAuth bridge.
  • Если в UniAuth нет сессии, пользователь проходит login/register/login.
  • После подтверждения происходит редирект обратно во внешний сервис с token3.
  • Callback внешнего сервиса вызывает introspect-token-3 с Bearer application API token.
  • При status=OK пользователь авторизован во внешнем сервисе локальным токеном.
  • Повторный вызов introspect с тем же token3 дает status=ERROR (одноразовость подтверждена).
